Local Canvas 是一个搭 AI 工作流的本地工具。你可以在白板上拖拽组合各种 AI 能力，做成一个自动化流程，一键运行。

它内置了一个轻量 AI 模型（Qwen2.5-3B），开箱即用，不需要配 Key、不需要联网、不需要装 Ollama。


━━━━━━━━━━━━━━━━━━━━━━━━━━
一、怎么打开
━━━━━━━━━━━━━━━━━━━━━━━━━━

⚠️ 注意：这个工具需要 Node.js 才能跑。如果你的电脑还没装过：

  1. 打开 https://nodejs.org
  2. 点左边的绿色大按钮（LTS 版本），下载安装包
  3. 双击运行，一路点"Next"，全部默认就行，不用改任何选项
  4. 装好后，回到这个文件夹

装好 Node.js 之后：

Windows 用户：双击文件夹里的 "start.bat"
Mac 用户：打开终端，cd 到项目目录，执行：
  chmod +x start.sh && bash start.sh

第一次运行会慢一些（它会自动下载依赖包），以后就快了。
等十几秒，会自动弹出一个浏览器页面，地址是 http://localhost:5173。
这就是 Local Canvas 的主界面了。


━━━━━━━━━━━━━━━━━━━━━━━━━━
二、界面长什么样
━━━━━━━━━━━━━━━━━━━━━━━━━━

打开后你会看到顶部有两个标签页：

  画布  ┃  设置

「画布」是主工作区，你大部分时间都在这里。
「设置」用来配置模型、API Key、知识库。

━━ 画布页面的布局 ━━

┌────────────┬──────────────────┬──────────────┐
│   左边     │     中间         │    右边      │
│  资源面板  │     画布         │   AI 对话    │
│ (Skills/   │  (拖拽节点搭     │  (用大白话    │
│  知识库/   │   工作流的地方)   │   指挥 AI)   │
│  模型/API) │                  │              │
└────────────┴──────────────────┴──────────────┘

左边：把可用的东西拖到中间白板上
中间：连线、搭流程
右边：跟 AI 聊天，用说的方式让它帮你干活


━━━━━━━━━━━━━━━━━━━━━━━━━━
三、配置 AI 模型
━━━━━━━━━━━━━━━━━━━━━━━━━━

Local Canvas 支持多种 AI 模型：

  内置模型（推荐，开箱即用）
  如果你有 models/builtin.gguf 文件，打开画布右侧 AI 面板会自动显示
  「内置模型 (本地)🟢 在线」，直接打字就能用，无需任何配置。

  外部模型（OpenAI / Ollama / Anthropic）
  点击顶部「设置」标签，进入「模型管理」：

  1. 点右上角的「添加」按钮
  2. 填下面这些信息：

     名称：随便起，比如 "我的GPT" 或者 "本地Ollama"
     适配器：选你用的模型类型
             OpenAI   → 如果你有 ChatGPT 的 Key
             Ollama   → 如果你本地装了 Ollama
             Anthropic→ 如果你用 Claude
     API 端点：填 API 地址
             OpenAI 填  https://api.openai.com/v1
             Ollama 填  http://localhost:11434/v1
     模型 ID：填具体的模型名，比如 gpt-4o 或 qwen3
     Key：填你的 API Key（OpenAI 的 sk- 开头的那个）

  3. 点「保存」

添加成功后，列表里会多一条，右边会显示绿色「在线」标记。

💡 小提示：你也可以到「API Key」标签页里单独管理你的 Key，方便重复使用。


━━━━━━━━━━━━━━━━━━━━━━━━━━
四、开始搭工作流
━━━━━━━━━━━━━━━━━━━━━━━━━━

回到「画布」标签页。

左边的资源面板有这些可以拖的东西：

  Skills    → 各种 AI 技能（文本摘要、翻译等）
  知识库    → 你之前配好的知识库
  模型      → 你配好的 AI 模型
  API       → 外部接口（天气、搜索等）

怎么搭一个流程：

  1. 从左边把你要用的节点拖到中间白板上
     比如拖一个「模型」节点、一个「Skill」节点

  2. 鼠标移到节点上，节点边上会出现小圆点
     点住小圆点拖到另一个节点上，就连上线了
     连线代表数据流向：上一个节点的输出 → 下一个节点的输入

  3. 点一下节点，右边会弹出配置面板
     可以在这里绑定模型、填参数等

  4. 点一下连线，可以设置数据映射
     比如"上游的输出字段 A"对应"下游的输入字段 B"

💡 常用快捷键：
  Ctrl+S   → 保存当前画布
  Ctrl+O   → 打开之前保存的工作流
  Ctrl+Z   → 撤销
  Ctrl+Y   → 重做


━━━━━━━━━━━━━━━━━━━━━━━━━━
五、运行工作流
━━━━━━━━━━━━━━━━━━━━━━━━━━

画布搭好后，点顶部工具栏的「运行」按钮（▶️ 图标）。

运行过程中：
  - 底部会弹出执行日志窗口，实时显示每一步的状态
  - 正在跑的节点会高亮
  - 跑完的节点会变绿
  - 出错的节点会变红

执行完后，可以在日志窗口看到完整的过程和结果。


━━━━━━━━━━━━━━━━━━━━━━━━━━
六、用 AI 对话辅助（核心功能）
━━━━━━━━━━━━━━━━━━━━━━━━━━

右侧的 AI 对话面板是 Local Canvas 的大脑，你可以用大白话指挥它做任何事。

内置模型开箱即用，无需配 Key。如果 models 目录下有 builtin.gguf 文件，
AI 面板会自动显示「内置模型 (本地)🟢 在线」。

━━ AI 能做什么 ━━

画布操作：
  "帮我加一个文本摘要节点"
  "把这两个节点连起来"
  "给节点 A 设置参数"
  "执行这个工作流"
  "清空画布"

文件操作（AI 可直接读取你电脑上的文件）：
  "帮我读一下 D:/work/report.txt"
  "列出 D:/文档 目录下有哪些文件"
  "帮我找一下桌面上所有的 PDF 文件"

模型管理：
  "帮我添加一个 OpenAI 模型"
  "列出所有可用的模型"

知识库管理：
  "添加一个知识库，路径是 D:/我的文档"
  "重建索引"

工作流管理：
  "帮我保存当前画布"
  "加载我之前保存的工作流"
  "导出这个工作流"

设置管理：
  "帮我保存一个 API Key"
  "帮我把 ID 为 3 的模型删掉"

━━ 对话框使用技巧 ━━

- 打字可以直接说，不用学命令格式
- 如果 AI 没有正确理解，换一种说法再问一次
- 对话框下方有快捷提示，点一下就能发送
- AI 会记住对话上下文，可以连续对话


━━━━━━━━━━━━━━━━━━━━━━━━━━
七、知识库（可选）
━━━━━━━━━━━━━━━━━━━━━━━━━━

如果你想让 AI 基于你自己的文件回答问题：

  1. 进入「设置」→「知识库」→ 点「添加」
  2. 名称：起个名字
  3. 文件夹路径：填你放文档的文件夹路径，比如 D:\我的文档
  4. 保存后，鼠标移到那条记录上，点「索引」
  5. 等索引跑完，知识库就可以在工作流里用了

索引完成后，你可以在画布上拖入知识库节点，让 AI 从你的文档里找答案。


━━━━━━━━━━━━━━━━━━━━━━━━━━
八、保存和加载
━━━━━━━━━━━━━━━━━━━━━━━━━━

  Ctrl+S  → 保存当前画布（第一次会提示起名字）
  Ctrl+O  → 弹出列表，选择之前保存的加载回来
  Ctrl+E  → 把当前工作流导出成文件

你也可以让 AI 帮你做这些事：说"帮我保存工作流"就自动保存了。


━━━━━━━━━━━━━━━━━━━━━━━━━━
九、常见问题
━━━━━━━━━━━━━━━━━━━━━━━━━━

Q: 打开是白屏怎么办？
A: 后端启动需要几秒，刷新一下浏览器就行。

Q: 内置模型一直显示「离线」？
A: 检查 models/ 目录下有没有 builtin.gguf 文件。如果没有，需要下载
   Qwen2.5-3B-Instruct-Q4_K_M.gguf（约 2.1GB）并重命名为 builtin.gguf
   放入 models/ 目录。

Q: 内置模型第一次回复很慢（40 秒左右）？
A: 正常。首次加载需要把 2.1GB 的模型读入内存。后续对话会快很多（5-15 秒）。

Q: 模型显示「离线」？
A: 检查 Key 有没有填对，API 地址能不能访问。

Q: 运行报错？
A: 看看执行日志窗口，里面会写明哪一步出了什么问题。

Q: 端口提示被占用？
A: 把之前开的黑窗口关掉，重新双击 start.bat。

Q: 下次怎么打开？
A: 直接双击 start.bat，比第一次快很多。

Q: Mac 上打开终端报 "Permission denied"？
A: 需要先给脚本加权限。执行一次 chmod +x start.sh，之后再运行 ./start.sh 就行。


━━━━━━━━━━━━━━━━━━━━━━━━━━
十、简单示例：搭一个翻译流程
━━━━━━━━━━━━━━━━━━━━━━━━━━

想试试手的话，跟着这个例子走一遍（不需要配置任何 Key，内置模型就能用）：

  1. 打开首页，右侧 AI 面板显示「内置模型 (本地)🟢 在线」
  2. 从左边把「模型」节点拖到白板上
  3. 点一下这个节点，在配置面板绑定内置模型
     参数里填：{ "prompt": "把以下内容翻译成英文: {{input}}" }
  4. 点工具栏的「运行」
  5. 在弹出的输入框里输入一句中文
  6. 等执行完成，看日志里的结果

或者直接用 AI 对话：

  在右侧对话框输入："帮我搭一个翻译工作流，把用户输入的中文翻译成英文"
  AI 会自动帮你完成以上所有步骤。

搞定！这就是一个最简单的 AI 工作流。
你可以继续拖更多节点、连更多线，组合出更复杂的流程。
