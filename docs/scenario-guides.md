# 场景教程

## 1. 翻译工作流

**目标：** 输入中文文本，AI 自动翻译成英文输出。

**步骤：**
1. 打开 Local Canvas，空白画布首次会自动加载「快速体验」模板
2. 点击工具栏「模板」按钮，选择「AI 翻译」
3. 双击「输入文本」节点，修改输入内容为你要翻译的文字
4. 双击「翻译」节点，确认模型已配置（需先在设置中添加 API Key 或 Ollama）
5. 点击工具栏「运行」按钮
6. 在弹出窗口中查看「日志」和「结果」

**技巧：** 修改翻译节点的 `temperature: 0.3` 可以获得更准确的翻译。

---

## 2. 知识库问答

**目标：** 索引本地文档，基于文档内容回答用户问题。

**步骤：**
1. 打开「设置」→「知识库」→「添加知识库」
2. 选择一个包含文档的文件夹（支持 PDF、TXT、Markdown 等）
3. 点击「索引」，等待索引完成
4. 返回画布，加载「知识库问答」模板
5. 选择「知识检索」节点，在配置中选择刚才创建的知识库
6. 运行工作流，输入问题查看结果

**技巧：** 配置 embedding 模型可以获得更准确的语义检索效果。

---

## 3. 数据清洗管道

**目标：** 输入 JSON 数据，通过 JavaScript 代码清洗后输出。

**步骤：**
1. 加载「数据清洗」模板
2. 拖拽一个「API 调用」节点到画布，配置 API URL
3. 连接：API → 清洗逻辑 → 统计结果
4. 运行，查看清洗后的数据

**技巧：** 使用 `file_output` 节点替换 `output` 节点可将清洗结果保存为 CSV/JSON 文件。

---

## 4. 网页内容分析

**目标：** 自动抓取网页 → AI 分析 → 输出结构化摘要。

**步骤：**
1. 加载「新闻聚合」模板
2. 在输入节点中填入要分析的 URL
3. 在「抓取内容」节点配置 API URL（自动使用输入节点的 URL）
4. 运行，获取分析结果

**技巧：** 串联多个 model 节点可以分步处理——第一步提取，第二步分类，第三步格式化。

---

## 5. 图片多模态分析

**目标：** 使用支持视觉的模型分析图片内容。

**步骤：**
1. 确保配置了支持 vision 的模型（如 GPT-4V、Claude 3、llava 等）
2. 加载「图片分析」模板
3. 在输入节点填入图片的本地路径或 base64 数据 URL
4. 选择「图片分析」节点，选择已配置的视觉模型
5. 运行，查看描述结果

**技巧：** 修改 prompt 可以指定分析方向，如"识别图片中的人物"、"提取所有文字"等。

## 常见工作流模式

### 线性管道 (Linear Pipeline)
```
输入 → 处理1 → 处理2 → 输出
```
最简单的结构，适合翻译、摘要等单步 AI 任务。

### 分支管道 (Branching)
```
         ┌→ 处理A ┐
输入 → ──┤        ├→ 输出
         └→ 处理B ┘
```
并行执行多个独立任务，如同时翻译成英文和日文。

### 检索增强生成 (RAG)
```
输入 → 知识检索 → AI 生成 → 输出
```
最强大的模式之一：先用本地知识库检索相关文档，再让 AI 基于检索结果生成回答。

### 代码嵌入 (Code Injection)
```
输入 → 代码处理 → AI 生成 → 输出
```
在 AI 处理前后插入自定义代码逻辑，实现数据转换、清洗、验证等功能。

### 条件分支 (Conditional Flow)
```
输入 → 条件判断 → True/False → 处理 → 输出
```
根据条件表达式决定数据流向，如仅处理评分>0.5 的内容。
